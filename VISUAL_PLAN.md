# 페이퍼 히어로즈 — 비주얼 개선 계획

> 출처: `chobabdefense.pdf`(= 페이퍼 히어로즈 간단기획서, 아트 디렉션 슬라이드 09 외) 정독 기반.
> 현재 프로토타입(프리미티브 캡슐/큐브 + 플랫 단색)을 기획서의 목표 비주얼로 끌어올리는 단계별 계획.

## 0. 현재 vs 목표

| | 현재(프로토) | 목표(기획서) |
|---|---|---|
| 캐릭터 | 캡슐/큐브 프리미티브 | **SD·복셀 로우폴리 종이공작**(귀여운 종이박스 + 얼굴, 종이무기) |
| 무드 | 단색 + 어두운 네이비 배경 | **어둡고 몽환적이지만 귀여운** 꿈/악몽, 보라·네이비 글로우 |
| 색 | 임의(탱=탠/근접=노랑/원=초록/힐=시안/적=보라) | **색 가이드**: 탱=파랑·근접=초록·원=노랑·힐=주황·적=빨강·거점=회색 |
| 이펙트 | 거점 파괴 파티클 1종 | 소환/피격/사망/힐 종이 연출 |
| UI | 나눔고딕 적용된 기본 버튼 | 덱처럼 둥근 카드 + 역할 아이콘 + 글로우 |

## P0 — 즉시(반나절, 코드/세팅만으로 최대 체감) ⭐ 추천 시작점
프리미티브 그대로 두고도 "데모"가 "몽환적 프로토"로 확 바뀌는 항목들.

1. **색 가이드 정렬** — `prototypeColor`만 기획서 가이드에 맞춤(탱=파랑, 근접=초록, 원=노랑, 힐=주황, 졸개/부기맨=빨강 계열, 거점=회색). 데이터 수정뿐, 즉시.
2. **URP 포스트프로세싱 볼륨** — Bloom(글로우) + Vignette + Color Grading(보라/네이비 틴트). 한 번에 "몽환" 무드.
3. **배경·조명 무드** — 카메라 배경을 보라→네이비 그라데이션(또는 그라데이션 스카이박스), 약한 안개, 캐릭터 림라이트. 디렉셔널 라이트 각도·색을 차갑게.
4. **전장(지면)** — 회색 큐브 → 종이/골판지·노트 질감 텍스처 평면 + 라인 가이드(거점 사이 경로 표시).
5. **유닛 실루엣 차별화** — 복셀 교체 전까지 프리미티브 스케일/형태로 역할 구분(탱커=납작한 박스, 근접=표준, 원거리=가늘고 큼, 힐러=작고 둥근).

## P1 — 게임필(피드백·이펙트)
6. **VFX**: 소환=종이 펼쳐짐, 피격=종이조각 파티클, 사망=구겨져 사라짐, 거점 타격=흔들림+조각, 힐=회복 글로우.
7. **카메라 셰이크**: 거점 피격·승패 순간 약한 셰이크.
8. (선택) 플로팅 데미지/힐 숫자.

## P2 — 아트 에셋 교체(복셀/SD, 토도님 협업)
9. **복셀 프리팹 교체** — 6종(종이박스 탱커·종이칼 근접·종이활 원거리·종이지팡이 힐러·졸개·부기맨). `UnitData`에 `prefab` 필드 추가 → `UnitSpawner`가 prefab 우선 사용(없으면 프리미티브 폴백). 코드 변경 최소.
10. **표정/종이 질감** — 캐릭터 얼굴(귀여움) + 종이 머티리얼.
11. **간단 애니메이션** — 이동 흔들림·공격·사망(종이 찢김/구겨짐).

## P3 — UI 리디자인(기획서 덱 미학 차용)
12. **유닛 버튼** — 둥근 카드 + 역할 아이콘(방패/검/활/지팡이) + 색 강조 + **쿨다운 라디얼/오버레이**(현재 텍스트 "쿨 N s" → 시각 게이지).
13. **자원 바** — 코인 아이콘 + 채워지는 바 + 증가 애니메이션.
14. **결과 패널** — 덱처럼 둥근 카드 + 글로우(현재 텍스트만), 성과/별점(선택).

## 우선순위 요약
- **P0**: 코드·세팅만으로 최대 임팩트 → **여기부터**. (색 정렬 + 포스트프로세싱 + 배경 무드)
- **P1**: 타격감(VFX·셰이크).
- **P2**: 복셀 아트 교체(아트 의존 — 데이터-로직 분리 덕에 코드 영향 최소).
- **P3**: UI 카드/아이콘.

## 설계상 유리한 점
- 데이터-로직 분리(`UnitData.prototypeColor` 등 SO)라 **색·수치는 코드 없이** 조정 가능.
- `UnitSpawner.SpawnUnit`이 단일 진입점 → 프리미티브→프리팹 교체가 한 곳에서 끝남.
- URP 사용 중 → 포스트프로세싱(Bloom 등) 바로 적용 가능.
